- Although not directly apparent from the code in udtools/validate.py (once in remake branch, now removed), I think
  that the idea was to also allow for custom checks on top of the official validator. While it is easy to derive a new
  class from Validator and define new check_...() methods in it, it is an open question how such extensions should be
  made available for the user through the command line interface. Ideally, they could be turned on and off the same way
  as the standard checks. But this can be left as a separate issue for the future.
